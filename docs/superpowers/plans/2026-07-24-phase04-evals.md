## The five evals (each maps to a review "silent on")

| # | Eval | Upstream (phase) | Metric | Pass bar |
|---|------|------------------|--------|----------|
| 1 | Promotion precision | triage gate (`promote`, 02) | precision & recall on admissions | precision ≥ 0.8, recall ≥ 0.8 |
| 2 | Skill fidelity | distilled `SKILL.md` vs trajectory (`improve`, 01) | judge faithfulness, mean over items | mean ≥ 0.8 |
| 3 | Skills-help | agent ± harvested skills (01) | success delta, with vs without | with ≥ without (and both scored) |
| 4 | Card fidelity over horizon | compaction (`context`, 03) | verbatim recall of spaced planted facts | recall ≥ 0.6, reported as distribution |
| 5 | Cost per correct answer | card / transcript / stateless (03) | cost per correct, with variance over trials | card ≤ transcript on cost-per-correct |

Every eval: build a labelled dataset → push to Langfuse (`create_dataset` + items) → run each item inside `item.run(...)` so the trace links to the dataset → `create_score` per item → aggregate locally → `check(...)` the aggregate → persist rows to `HARNESS_EVAL`.

The `item.run()` / `start_as_current_span` shapes do **not** exist here. The verified, pinned API — which the generator MUST use verbatim — is the **Experiments** API:

```python
from langfuse import get_client, Evaluation
lf = get_client()
assert lf.auth_check()

lf.create_dataset(name="promotion-precision")                       # idempotent by name
lf.create_dataset_item(dataset_name="promotion-precision",
                       input={...}, expected_output={...}, metadata={...})
ds = lf.get_dataset("promotion-precision")

def task(*, item, **kwargs):                # receives DatasetItem: item.input/.expected_output/.metadata
    return {...}                            # the arm's output for this item

def score(*, input, output, expected_output, metadata, **kwargs) -> Evaluation:
    return Evaluation(name="admitted_correctly", value=1.0 or 0.0, comment="...")

res = ds.run_experiment(name="promotion-precision", task=task, evaluators=[score])
# res.item_results[i] -> .output, .evaluations (list of Evaluation), .trace_id, .dataset_run_id
# res.dataset_run_url, res.run_evaluations, res.format(include_item_results=True)
lf.flush()
```

Confirmed against the live server: the experiment run is created and linked, per-item traces carry the evaluator scores, and server-side average scores are computed. `run_experiment` auto-creates the dataset run — there is no separate `create_dataset_run`. Aggregate metrics for the in-notebook `check(...)` are still computed locally from the same per-item data (the notebook does not read scores back from the server).
