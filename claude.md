# cityops-agent-harness

A 4-notebook series (package `cityops_harness`) teaching agent memory + harness
engineering — the sequel to the CityOps Copilot workshop. The original workshop
notebook `notebook_complete_ollama.ipynb` must stay untouched.

foundation → 01 self-improving copilot → 02 scheduled briefings → 03 context
engineering → 04 evals.

## Key decisions (do not re-litigate)

- OCI Autonomous DB 23ai Always Free; in-DB ONNX embeddings. `EMBED_MODEL` stays
  `ALL_MINILM_L12_V2` — cross-notebook state checks in `cityops_harness/state.py`
  reference it statically.
- All three LLM providers first-class (Ollama / Anthropic / OpenAI) via one
  factory in `cityops_harness/llm.py`.
- Langfuse supports cloud and self-hosted (local) modes; `LANGFUSE_MODE=off` is valid.
- Each notebook is authored as `*_complete.ipynb`; learner `*_todo.ipynb`
  versions are generated by `tools/make_*_notebook.py`-style generators.
- Implementation closes the promotion gaps in `memory_promotion_review.md`
  (curation gate, verified success, skill lifecycle, provenance/supersession);
  the 5 Langfuse evals in notebook 04 target that review's "silent on" column.
- Live-calibrated retrieval thresholds (notebook 01): TOOL_DISTANCE_MAX=0.92,
  SKILL_DISTANCE_MAX=0.65.
- Naming: professional/descriptive names, not whimsical.

## Environment

- Local dev: `uv`-managed Python 3.12 venv at `.venv/`.
- Secrets come from `.env` locally (git-ignored) and from Codespaces secrets in
  a Codespace: WALLET_B64, DB_USER, DB_PASSWORD, WALLET_PASSWORD,
  ANTHROPIC_API_KEY. Never commit credentials.
- DB wallet lives in git-ignored `wallet/` (locally, decoded from WALLET_B64 in
  Codespaces by post-create).
- One-time DB grant already applied: `GRANT EXECUTE ON DBMS_CLOUD TO ANANT`
  (run as ADMIN; documented in README).
